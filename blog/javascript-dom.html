An analogy to describe the differences between HTML and CSS

HTML and CSS are the core building blocks to building a website. Lets imagine you are wanting to start a cafe and you need to set up the layout of the cafe. You have a list of things you will need for the cafe and so you start gathering these things into suitable locations. You will need seats and tables, a counter to order at, a space for the kitchen and a space for the coffee machine. These features are organizing the cafe just as HTML organizes a webpage with elements to set up the structure.

This is great but what have we envisioned for our cafe? We want to open up a cafe that looks fantastic and people want to come and enjoy a coffee in a nice environment. This also takes planning, else we will be left with a run of the mill cafe. CSS is used to interact with the HTML document to style the webpage. We know we need a coffee machine but what do we want our coffee machine to look like? Using CSS is how we get the webpage to be designed as we would like.

Control flow and loops

Control flow in Javascript works by executing code in a path from top to bottom unless there is a condition set. Such a condition may be an If , Else condition that tells the code hold on a second we have a couple options here due to some defined parameters. An example of this is at our cafe we have customers who come to order. It would be easy to offer them all the same coffee, say a long black. However some people do not drink black coffee. IF the customer wants a white coffee serve them a (‘Flat White’) Else If the customer wants a black coffee serve them a (‘Long Black’) Else (they don’t like coffee). A loop is a great way to execute some code without having to repeatedly state it. If we had a regular customer and we know what their coffee order is we can then set this data into an array and use a For condition to loop through when this customer enters the cafe. For ‘Sam’ make an Americano and we don’t have to ask him what he would like each time.

Arrays and objects

Arrays and objects are both used for holding data in Javascript. An array is an ordered collection of items which are stored in numerical position using “zero-based indexing” meaning that the first item in an array is actually in position 0. Here is what an Array looks like with a group of strings.

let array = [“this”, “is”, “a”, “string”]

To access the first item in an array it would look like this.

const firstItem = array[0].

Objects are used to store groups of data rather than single data values. An object could be anything that has a set of characteristics. An object is laid out like this:

let object = {hat: ‘green’, top: ‘pink’, shorts: ‘black’ }

and to access an object we can use dot notation or bracket notation.

const objectGreen = object.green | const objectGreen = object[‘green’]

Functions

A function is a block of code that we use to perform tasks. A function can perform a single task or multiple tasks.

function myFunction(p1, p2) { return p1 * p2;}

If we enter myFunction with two data points it will return to us p1 multiplied by p2. The great thing about functions is once the function has been set up it can be executed as many times as wanted to keep performing a task, this is massively time saving.
